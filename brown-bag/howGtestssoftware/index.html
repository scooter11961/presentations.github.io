Testing Techniques

How Google test[ed] Software

Scott Hatch

Does this model fit us?

After reading HGTS, it became obvious to me that Google is a large enough company; with a complex enough product line that one model does NOT fit all cases.

In general, the web product teams (like gMail, Google plus, gMaps, etc.) are similar to us, and those are the examples I drew comparisons with. Others that are more different than similar included youTube (slight overlap), Chrome (slight overlap) and Android (OS/Apps side, but not hardware side)

How Google Teams are organized:

  • SWE (Software Engineers)
  • SET (Software Engineers in Test)
  • TE (Test Engineers)
  • very few designers
  • some Product Owners
  • NO Quality Assurance Team

Why does this work for them?

  • Googlers use their own products daily!
  • They are highly skilled
  • They have 20% time to reinvest in learning / contributing

SWE (Software Engineers)

  • Works in a test driven development model
  • Creates unit tests for code, javascript, etc.
  • Creates spec tests that may test with browsers
  • They may build mocks or fakes for unit tests to interact with
  • SWEs own the quality of the product they build, there is no 2nd team acting as a QA last check safety net.
  • SWEs fix the bugs that are found in their products
  • They submit ‘change sets’ of code
  • They perform code reviews

Does MapQuest have SWE's ?

Yes

This maps very closely to our idea of Software Engineers. Basic responsibilities include feature story interpretation, code architecture, functional code creation, product architecture, and health monitoring.

SET (Software Engineers in Test)

  • SETs work to make SWEs more efficient.
  • Setup Agile tools like build servers (Jenkins), deploy scripts (shell scripts, Capistrano scripts, test frameworks (like rspec/capbybara/selenium) javascript test frameworks (jasmine)
  • They also contribute to building, reviewing, and extending unit tests
  • They likely build mocks and fakes for unit tests to interact with
  • They may build custom tools when needed (java/ruby test code?)
  • They may contribute to functional code occasionally

Does Mapquest have SET's ?

No

Not presently dedicated full time. We do have more technically oriented test engineers than some. We do have Software Engineers that take the 'lead' on implementing 'best practices' and tools for the benefit of the team, and contributing to a better quality software development process.

TE (Test Engineers)

  • TEs are responsible for the overall width of testing and coverage
  • Contribute to automated tests
  • Perform risk analysis of features / products in order to guide the team in where to test, and how much to test based on what is most important
  • Perform organized, exploratory testing

Does MapQuest have TE's ?

Yes
